## Summary
jq is a command-line tool that slices, filters, maps, and transforms structured JSON using a compact expression language. It reads JSON on stdin (or from a file), applies a filter program, and writes the result — making JSON pipeable like text through standard Unix tooling.

## Why installed
APIs, config files, and CLI tools (including brew, npm, and Docker) all speak JSON. jq turns that output from an unreadable blob into something you can pretty-print, drill into, and extract single values from — the missing piece for scripting against any JSON-emitting command.

## Most common use case
Piping a command's JSON output into jq to pretty-print it or pull out specific fields — for example, extracting one value from a `curl` API response or filtering an array of objects.

## Biggest win
It makes JSON a first-class citizen of the shell pipeline. Instead of writing a throwaway script to parse a response, a one-line jq filter selects, reshapes, and reformats the data inline — composing cleanly with every other CLI tool.

## How to use
```sh
# Pretty-print JSON
cat data.json | jq .
curl -s https://api.example.com/items | jq .

# Extract a field / nested path
jq '.name' data.json
jq '.user.email' data.json

# Map over an array, select matching objects
jq '.items[].id' data.json
jq '.items[] | select(.active == true)' data.json

# Reshape into a new object
jq '{id: .id, label: .name}' data.json

# Raw string output (no quotes), useful in scripts
jq -r '.version' package.json
```

## Future use
Build reusable jq filter files (`jq -f filter.jq`) for recurring transforms, and combine jq with `--arg`/`--argjson` to inject shell variables into queries — turning ad-hoc one-liners into maintainable data-shaping scripts.
