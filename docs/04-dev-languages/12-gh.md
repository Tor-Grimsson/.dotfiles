## Summary
`git` talks to *any* git remote; **`gh` talks to GitHub specifically** — the layer git doesn't cover. Pull requests, issues, reviews, releases, Actions runs, and the raw API, all from the terminal so you stop bouncing to the browser.

| Command | Does |
|---|---|
| `gh pr create` / `gh pr checkout <n>` | open a PR / check out someone else's PR branch |
| `gh pr view --web` / `gh pr merge` | open the PR page / merge it |
| `gh issue create` / `gh issue list` | file and browse issues |
| `gh run watch` / `gh run view --log` | follow a CI run / read its failure log |
| `gh api <endpoint>` | authenticated REST/GraphQL call (pipe to `jq`) |

## Setup
1. Install: `brew bundle` (now in the Brewfile) — or `brew install gh`.
2. Authenticate: `gh auth login` → pick **GitHub.com → HTTPS** → *Login with a web browser*, paste the one-time code. Stores a token locally (in the system keychain), **not** in the repo.
3. Verify: `gh auth status` and `gh repo view --web` from inside any repo.

> The oh-my-zsh **`gh` plugin** is already in `plugins=(…)` (`shell/.zshrc:9`) — it loads gh's completions. It was a no-op until now (no binary); installing gh makes it live.

## Use

```sh
gh pr create --fill                  # PR from the current branch, title/body from commits
gh pr status                         # PRs relevant to you (yours, review-requested)
gh pr checkout 42                    # check out PR #42's branch locally to test it
gh pr merge 42 --squash --delete-branch
gh issue create -t "title" -b "body"
gh issue list --label bug
gh run list                          # recent Actions runs
gh run watch                         # live-follow the latest run until it finishes
gh run view --log-failed             # just the failed step's logs
gh release create v1.2.0 ./dist/*    # cut a release + upload assets
gh repo clone owner/repo             # shorthand clone
gh browse                            # open the current repo/file/line in the browser
gh api repos/{owner}/{repo}/issues --jq '.[].title'   # script anything GitHub exposes
```

## Why installed
The GitHub-specific half of the daily git workflow: creating and reviewing PRs, reading CI failures, and scripting via `gh api` — none of which plain `git` does. Also resolves the dangling `gh` oh-my-zsh plugin that was loading completions for a missing binary.

## Biggest win
`gh pr checkout <n>` + `gh run watch` — pull a PR branch to test it locally in one command, then follow its CI to green without ever opening a browser tab. For automation, `gh api … --jq` is a fully-authenticated GitHub API client with no token plumbing.

## Note on auth vs this repo's sync
Dot-sync pushes over **SSH with a keychain'd key** (`ssh/config` `Host github.com`), so you don't need gh for `git push`. `gh auth login` is still required for the `gh` commands themselves (PRs/issues/API). gh *can* also act as a git credential helper for HTTPS remotes if you ever want that — not needed here.

## Reproduce on the other machine
`brew bundle` installs it; then `gh auth login` once per machine (the token is local, not synced).

## Future use
`gh pr review` for inline reviews from the terminal; `gh alias set` for custom shortcuts; `gh extension` for community subcommands; `gh api graphql` when REST isn't enough.
