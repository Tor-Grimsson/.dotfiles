## Summary

Finds files and directories by **name** — the friendly `find` replacement. Fast, colored, and it skips `.gitignore`d files and `.git` by default. Set as `FZF_DEFAULT_COMMAND` in `shell/.zshrc`, so it's what feeds every fzf file list.

## How to use

```sh
fd report                # names containing "report", here and below
fd -e pdf                # by extension
fd -e jpg -e png         # multiple extensions
fd todo ~/dev            # search starting in ~/dev
fd -t d photos           # directories only
fd -H -I secret          # include hidden + gitignored
fd -e log -x rm          # run a command on each match
```

| Flag | Does |
|---|---|
| `-e EXT` | filter by extension |
| `-t f\|d\|l` | type: file / dir / symlink |
| `-H` | include hidden |
| `-I` | don't respect `.gitignore` |
| `-x CMD` | exec CMD per result |
| `-g GLOB` | glob match instead of substring |
| `--strip-cwd-prefix` | drop the leading `./` |

## Why installed

`find`'s syntax is famously hostile (`find . -name '*.pdf'`); fd is `fd -e pdf`. It's also the fzf file source (`FZF_DEFAULT_COMMAND='fd --type f --hidden --strip-cwd-prefix --exclude .git'`), so every `Ctrl-T` and bare `fzf` is fd underneath. See [[12-fzf|fzf]].

## Biggest win

gitignore-aware by default — searches skip `node_modules`/`.git`/build junk with zero config, which is also what keeps fzf fast in big trees.

## Future use

`fd … -x` batch pipelines (rename/convert/delete swaths of files); ad-hoc `fd | fzf` pickers inside scripts.
