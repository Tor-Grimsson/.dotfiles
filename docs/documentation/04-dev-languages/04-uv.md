## Summary
uv is a single Rust binary that replaces most of the Python packaging stack — `pip`, `venv`, `pip-tools`, `pyenv`, and parts of `pipx`. It manages projects with a lockfile, resolves and installs dependencies extremely fast, and can download and pin Python interpreters itself.

## Why installed
It is the modern entry point for Python work here. Instead of juggling a separate version manager, a venv tool, and pip, uv does all three from one fast command — and it can provision the Python interpreter too, so the system Python stays untouched.

## Most common use case
Spinning up and running a project: `uv init`, `uv add <package>`, and `uv run <script>`, which transparently creates the virtual environment, resolves dependencies against the lockfile, and executes against it.

## Biggest win
Speed and consolidation. Resolution and installs are an order of magnitude faster than pip, and one tool covers interpreters, environments, and dependencies with a reproducible `uv.lock` — eliminating the usual pile of overlapping Python tools.

## How to use
```sh
# Start a project
uv init myproject && cd myproject

# Add / remove dependencies (updates pyproject.toml + uv.lock)
uv add requests
uv remove requests

# Run code inside the managed environment
uv run main.py
uv run python -c "import requests; print(requests.__version__)"

# Manage Python interpreters
uv python install 3.13
uv python list

# Sync an existing project's locked deps
uv sync
```

## Future use
Use `uv tool install` / `uvx` to run and manage standalone CLI tools (overlapping with pipx), adopt `uv.lock` across all Python projects for reproducible builds, and lean on `uv python pin` to lock each project to a specific interpreter version.
