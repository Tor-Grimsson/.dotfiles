# R2 bucket

One **Cloudflare R2** bucket, `kol-media`, driven by the
[`bucket-r2`](../../claude/packages/kol-cdn/kol-bucket-r2/kol-bucket-r2) CLI. Unlike B2 it is **not** rclone: reads go
through a public admin API, writes through Wrangler's OAuth session.

## Bucket

| Bucket | Read API | Public CDN | Writes |
|--------|----------|-----------|--------|
| `kol-media` | `https://admin.kolkrabbi.io/api/list` | `https://media.kolkrabbi.io` | `wrangler r2 object put/delete --remote` (OAuth) |

The admin API + bucket are the [[documentation/15-cloudflare/INDEX|`kol-media-admin`]] project
(Cloudflare Pages + Functions) — its web UI is the skin over this same bucket.

## How to use

```sh
bucket-r2 ls  type/              # list under a prefix (public API, no auth)
bucket-r2 url labs/x.png         # public CDN URL
bucket-r2 cat labs/x.png         # stream a file
bucket-r2 up  ./x.png labs/x.png # upload  (wrangler OAuth; fires a tree snapshot)
bucket-r2 rm  labs/x.png         # delete  (fires a tree snapshot)
```

No bulk/recursive upload in the wrapper — a whole folder goes through
`kol-media-admin/scripts/bulk-upload.sh`. Full detail in the
[`kol-bucket-r2`](../../claude/skills/kol-bucket-r2/SKILL.md) skill.

## Where it's used / consumers

- **kol-system tools** and the `kol-media-admin` web UI serve media from `media.kolkrabbi.io`.
- Anything embedding a `media.kolkrabbi.io/...` URL reads this bucket.

## Snapshots

`bucket-tree.sh r2` walks the admin list API (**paginated** — loops on `cursor`) and regenerates:
- [[04-r2-tree|`04-r2-tree.md`]] — readable tree view (Obsidian)
- `_files/kol-media/{tree.json, tree.full.txt}` — raw (nvim / machines)

A `bucket-r2` write refreshes them automatically (the [[05-scripts|post-write hook]]); the
wrangler OAuth session must be live for a write (and thus the hook) to fire.
