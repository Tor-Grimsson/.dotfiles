## Summary
w3m is a classic text-mode browser: no JavaScript, no images by default, but instant startup, proper table rendering, colors, and mouse support. Beyond interactive browsing it doubles as an HTML-to-text renderer for pipelines via `-dump`.

## Why installed
The zero-dependency daily-driver for text content, and the fallback that keeps working without Docker or OrbStack running. [Carbonyl](01-carbonyl.md) handles the modern web; w3m handles everything that's just text — faster.

## Most common use case
```sh
w3m https://kolkrabbi.io
```

## Biggest win
`-dump` — render any page to plain text on stdout and pipe it anywhere: `w3m -dump https://example.com | grep …`.

## How to use
**The syntax gotcha:** in `usage: w3m [options] [URL]` the square brackets mean *optional* — never type them. Worse, zsh treats `[...]` as a glob pattern, so `w3m [url]` fails with the misleading `zsh: no matches found`. Also `-t tab` sets tab *width* (a number), not "open in tab".

Keys inside w3m:

| Key | Action |
| --- | --- |
| arrows | move cursor |
| `Tab` | jump to next link |
| `Enter` | follow link |
| `B` | back |
| `U` | open a new URL |
| `/` | search |
| `o` | options panel |
| `H` | help |
| `q` | quit |

Useful flags: `-dump` (render to stdout), `-dump_source` (raw HTML), `-cols 100` (wrap width with `-dump`), `-no-mouse`.

## Config
Stored in `~/.w3m/config`; edit interactively via the `o` options panel (saved automatically). Not symlinked into the dotfiles — defaults are fine.

## Future use
Poor-man's readability pipeline: `w3m -dump url` chained into grep/less/glow-style readers.
