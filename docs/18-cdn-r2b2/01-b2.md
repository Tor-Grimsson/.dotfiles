# B2 buckets

Two **Backblaze B2** buckets, both driven by the [`bucket`](../../claude/packages/bucket) CLI
(a thin rclone wrapper). Public read over HTTPS; writes via rclone with the `[kolkrabbi]` key.

## Buckets

| Bucket | rclone remote | Public CDN base |
|--------|---------------|-----------------|
| `website` | `kolkrabbi:kolkrabbi/website` | `https://f005.backblazeb2.com/file/kolkrabbi/website` |
| `vault-media` | `kolkrabbi:kol-vault-media` | — (private-ish; vault attachments) |

`website` lanes: `art-prints/` · `asset-library/` · `hls-library/` (HLS video) · `data-library/`.

## How to use

`bucket` defaults to the `website` lane; reach `vault-media` with the env override.

```sh
bucket ls   hls-library/video-library      # list a path
bucket tree art-prints                      # tree view (drops segment_*.ts)
bucket url  asset-library/x.jpg             # public CDN URL
bucket up   ./local  hls-library/foo        # upload  (fires a tree snapshot)
bucket rm   asset-library/old.jpg           # delete  (fires a tree snapshot)

BUCKET_REMOTE=kolkrabbi:kol-vault-media bucket ls lobby   # the other bucket
```

Full command set + the destructive `sync` in the [`kol-bucket-b2`](../../claude/skills/kol-bucket-b2/SKILL.md) skill.

## Where it's used / consumers

- **`website`** — the public site's media: art prints, the asset library, and **HLS video**
  (e.g. `hls-library/video-library/work/` for the kol-monorepo work pages). Read by
  `kol-monorepo`, the foundry, and the R2/B2 snapshot consumers.
- **`vault-media`** — the Obsidian **kol-vault**'s media offload: `tg-inbox` captures land in
  `lobby/` and the vault embeds them by CDN URL (binaries never touch git).

## Snapshots

`bucket-tree.sh b2` lists both buckets and regenerates:
- [`02-b2-tree.md`](02-b2-tree.md) — readable tree view (Obsidian)
- `_files/{website,vault-media}/{tree.json, tree.full.txt}` — raw (nvim / machines)

A `bucket` write refreshes them automatically (the [post-write hook](05-scripts.md)).
